## 6. Security Considerations

### Impersonation

- **Endpoint:** `POST /hod/test-personas/:pupilId/impersonate`
- **Access:** Requires HoD role (`requireHoD` middleware)
- **Scope:** Only works for users where `is_test_persona = true`
- **Token:** 1-hour expiry (vs 30-day for normal login)
- **Audit:** Logged to `audit_log` table with action `impersonate_test_persona`
- **Client:** Original HoD token preserved in `fam_original_token`; amber banner
  visible during impersonation; "Return to HoD" restores original session

### Reset Protection

- `protected_from_reset = true` on both user and pupil records
- `wipeDemoTenant()` excludes protected users from deletion
- Protected pupils' `team_id` set to NULL before school cascade to prevent
  `ON DELETE CASCADE` from destroying their records
- `insertTestPersona` is idempotent: detects existing protected users by email
  and re-links them instead of duplicating

### Data Isolation

- Test personas are created as `is_demo_user = true` with `demo_expires_at`
- They exist only within the demo tenant (Ashworth Park Academy)
- No real student data is used; all content is fictional
